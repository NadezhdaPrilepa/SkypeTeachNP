Преимущества JAVA:

1) Кроссплатформенность.
Раньше под каждую платформу(windows, Linux,мобильные платформы ) необходимо было писать
свою программу (одна и та же программа).Sun Systems создала инерпритатор и язык,
который компилируется не в исполняемый код, а в байт код, который в свою очередь
упаковывается в jar. Интерпритатор транслирует байт код в исполняемые машинные команды (для windows ассемблер)
Т.о. для различных платформ (разные интерпритаторы) можно использовать одну программу(compile once, run anywhere(с))

2)Работа с памятью.
До этого программисту приходилось самому следить и выделении памяти,
в связи с этим возникали проблемы с утечкой памяти.
В JVM работает фоновый процесс GABARGE COLLECTOR, который помечает ненужные объекты и затем их удаляет.

3) HotSpot.
Это JVM, которая собирает статистику о программе, анализирует и влияет на
исполнение программы(повышает производительность).
Этим интерпритатор JVM лучше компилятора.

Минусы:
Вытекают из плюсов
Фоновый процесс GABARGE COLLECTOR может тормозить выполнение программы.
